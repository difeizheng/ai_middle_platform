### 能力差距分析

通过与主流 AI 平台（Dify、LangChain、Azure AI、AWS Bedrock）对比，识别以下差距：

1. **商业化能力不足** - 缺少计费系统、配额管理、成本分析
2. **用户体验待提升** - 缺少可视化编排器、统一前端门户
3. **安全合规待加强** - 内容审核、数据隐私保护、Prompt 注入检测
4. **协作功能缺失** - 团队工作空间、资源共享、评论批注
5. **运维能力待完善** - 多环境管理、灰度发布、链路追踪

## 二、Phase 5 商业化运营（近期优先级高）

### 5.1 计费系统

#### 功能描述
实现灵活多样的计费模式，支持商业化运营。

#### 子功能
| 功能 | 描述 | 优先级 |
|------|------|--------|
| 计费策略配置 | 按 Token/按调用次数/包月包年/阶梯定价 | P0 |
| 账户余额管理 | 充值、消费、退款、账单 | P0 |
| 实时计费 | API 调用时实时扣费 | P0 |
| 预算预警 | 余额不足时告警通知 | P1 |
| 计费报表 | 收入统计、消费分析、趋势预测 | P1 |

#### 数据模型设计
```sql
-- 计费策略表
CREATE TABLE billing_plans (
    id VARCHAR(36) PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    billing_type VARCHAR(20),  -- token/call/subscription
    price_per_1k_tokens DECIMAL,
    price_per_call DECIMAL,
    monthly_fee DECIMAL,
    quota_limit INTEGER,
    overage_rate DECIMAL,
    is_active BOOLEAN DEFAULT TRUE
);

-- 账户表
CREATE TABLE accounts (
    id VARCHAR(36) PRIMARY KEY,
    user_id VARCHAR(36) REFERENCES users(id),
    balance DECIMAL DEFAULT 0,
    total_recharge DECIMAL DEFAULT 0,
    total_consumption DECIMAL DEFAULT 0,
    currency VARCHAR(10) DEFAULT 'CNY',
    status VARCHAR(20) DEFAULT 'active'
);

-- 计费记录表
CREATE TABLE billing_records (
    id VARCHAR(36) PRIMARY KEY,
    account_id VARCHAR(36),
    record_type VARCHAR(20),  -- charge/consume/refund
    amount DECIMAL,
    balance_before DECIMAL,
    balance_after DECIMAL,
    resource_type VARCHAR(20),  -- model_call/knowledge_base/agent
    resource_id VARCHAR(36),
    tokens_used INTEGER,
    description TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 充值订单表
CREATE TABLE recharge_orders (
    id VARCHAR(36) PRIMARY KEY,
    account_id VARCHAR(36),
    amount DECIMAL,
    payment_method VARCHAR(20),
    payment_status VARCHAR(20),
    transaction_id VARCHAR(100),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### API 设计
```
POST   /api/v1/billing/plans              # 创建计费方案
GET    /api/v1/billing/plans              # 获取计费方案列表
POST   /api/v1/billing/accounts           # 创建账户
GET    /api/v1/billing/accounts/{id}      # 获取账户信息
POST   /api/v1/billing/accounts/{id}/recharge  # 充值
GET    /api/v1/billing/records            # 获取计费记录
GET    /api/v1/billing/stats              # 计费统计
```
